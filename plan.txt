#### Step-by-Step Implementation on Render (Easiest Start)
1. **Sign Up & Verify**: Create free Render account (GitHub login); no card needed.
2. **Deploy Web Service**: Connect your repo > New > Web Service > Select Next.js blueprint. Set env vars (GEMINI_KEY, PINECONE_API_KEY, etc.). Auto-builds with `npm run build`.
3. **Add Background Worker**: New > Background Worker > Point to your `/api/process-batch` handler. Configure as a service (runs on demand via API trigger from upload endpoint).
4. **Handle Large PDFs**: 
   - Upload to Render Disks (free 100GB) or external Blob (e.g., Azure free via Pack).
   - In upload route: Trigger worker via Render's internal API (`fetch('https://your-worker.onrender.com/process-batch', { body: { startPage: 1 } })`).
   - Use pdf-lib for subset extraction (as in prior code); stream Gemini via `generateContentStream`.
5. **Progress & UI**: Redis pub/sub for real-time updates; add UptimeRobot (free) to ping `/health` every 14min to prevent sleep.
6. **Quant Extensions**: In worker, post-extraction: Run GARCH sims (via numpy.js or offload to free Colab for demo vids). KG build: Async Neo4j upsert after all batches.
7. **Test & Optimize**: Deploy sample; monitor via Render Dashboard (free logs/metrics). For Hindi OCR: Gemini prompt includes "Transliterate Devanagari."

For Azure: Redeem Pack credit > Create Function App (free tier) > Deploy via VS Code extension. Use Durable Orchestrations for batch chaining—no timeouts.
